## GitHub Secrets

在 GitHub 仓库中设置：

```text
Settings -> Secrets and variables -> Actions -> New repository secret
```

需要添加：

```text
DEEPSEEK_API_KEY
SMTP_HOST
SMTP_PORT
SMTP_USERNAME
SMTP_PASSWORD
MAIL_FROM
MAIL_TO
SMTP_SSL
```

说明：

- `DEEPSEEK_API_KEY`：你的 DeepSeek API key。
- `SMTP_HOST`：发信服务器地址。
- `SMTP_PORT`：常用 `587` 或 `465`。
- `SMTP_USERNAME`：发信邮箱账号。
- `SMTP_PASSWORD`：发信邮箱密码或应用专用密码。
- `SMTP_SSL`：如果使用 465 端口，通常填 `true`；如果使用 587 STARTTLS，可填 `false` 或留空。
- `MAIL_FROM`：邮件发件人地址。
- `MAIL_TO`：邮件收件人地址，例如 `you@example.com`。

## 保密机制

- 不要把 API key、SMTP 密码、邮箱 token 写入任何文件。
- 不要把密钥写进命令行参数。
- GitHub workflow 会在运行前扫描疑似密钥。
- 脚本会拒绝从 `config.json` 读取 `api_key`、`token`、`password` 等字段。
- `.env`、`*.key`、`*.pem`、`secrets.*` 已被 `.gitignore` 忽略。
